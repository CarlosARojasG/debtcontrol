🔥 Configurar Firebase

Guía paso a paso para sincronizar DebtControl Pro con la nube (gratis)

1 Crear cuenta y proyecto en Firebase

Firebase es un servicio gratuito de Google para almacenar datos en la nube.

  • Ve a console.firebase.google.com
  • Inicia sesión con tu cuenta de Google
  • Click en "Crear un proyecto" (o "Add project")
  • Nombre del proyecto: escribe algo como debtcontrol
  • Desactiva Google Analytics (no se necesita) → Click "Crear proyecto"
📱 → 🌐 console.firebase.google.com
💡 El plan gratuito (Spark) incluye 1GB de almacenamiento y 10GB de transferencia mensual — más que suficiente para uso personal.
2 Crear la Realtime Database

Esta es la base de datos donde se guardarán tus datos.

  • En el menú izquierdo, busca "Compilación" (o "Build")
  • Click en "Realtime Database"
  • Click en "Crear base de datos"
  • Selecciona la región más cercana (ej: us-central1)
  • ⚠️ IMPORTANTE: Selecciona "Iniciar en modo de prueba" (test mode)
  • Click en "Habilitar"
🗄️ Compilación → Realtime Database → Crear
⚠️ Sobre el modo de prueba: las reglas de prueba expiran en 30 días. Cuando expiren, ve a la pestaña "Reglas" de tu database y reemplaza el contenido con esto:
{ "rules": { "users": { "$uid": { ".read": true, ".write": true } } } }

Esto permite leer y escribir dentro de /users/. Para mayor seguridad, puedes usar un PIN de cifrado dentro de la app.

3 Registrar una app Web

Esto genera las credenciales que DebtControl necesita para conectarse.

  • En el menú izquierdo, junto a "Información general del proyecto", hay un ícono ⚙️ → click en "Configuración del proyecto"
  • Baja hasta la sección "Tus apps"
  • Click en el botón </> (ícono de Web)
  • Nombre de la app: escribe DebtControl
  • NO marques "Firebase Hosting"
  • Click en "Registrar app"
⚙️ Configuración → Tus apps → </> Web
4 Copiar la configuración

Después de registrar la app, Firebase te mostrará un bloque de código. Busca el objeto firebaseConfig:

const firebaseConfig = { apiKey: "AIzaSyB1234567890abcdefg", authDomain: "debtcontrol-abc12.firebaseapp.com", databaseURL: "https://debtcontrol-abc12-default-rtdb.firebaseio.com", projectId: "debtcontrol-abc12", storageBucket: "debtcontrol-abc12.appspot.com", messagingSenderId: "123456789", appId: "1:123456789:web:abcdef123456" };
📋 Copia TODO este bloque (desde { hasta }). Lo necesitarás en el siguiente paso.

Si ya cerraste esta ventana, puedes volver a ver la config en: Configuración del proyecto → Tus apps → Config

5 Pegar la configuración en DebtControl

¡Último paso! Ahora conecta la app con tu Firebase.

  • Abre DebtControl Pro en tu navegador o celular
  • Toca el botón ☁️ (esquina inferior derecha)
  • Toca "⚙️ Configurar Firebase"
  • Pega el JSON copiado en la caja de texto y toca "📋 Parsear JSON"
  • Los campos se llenarán automáticamente
  • Toca "💾 Guardar y Conectar"
✅ Si ves el indicador verde en el botón ☁️, ¡estás conectado! Ya puedes usar "Subir a la Nube" y "Descargar de la Nube".

📱 Sincronizar entre dispositivos

A Dispositivo principal
  • Configura Firebase (pasos 1-5 de arriba)
  • Toca ☁️ → ⚙️ Configurar Firebase
  • Copia tu "ID de sincronización"
  • Sube tus datos: ☁️ → "⬆️ Subir a la Nube"
B Segundo dispositivo
  • Abre DebtControl Pro
  • Configura Firebase con los mismos datos (mismo proyecto)
  • Toca ☁️ → ⚙️ → "✏️ Cambiar ID"
  • Pega el ID del dispositivo principal
  • Toca ☁️ → "⬇️ Descargar de la Nube"
✅ ¡Listo! Ambos dispositivos comparten los mismos datos.

🔒 PIN de cifrado (opcional)

Si quieres que nadie más pueda leer tus datos en Firebase:

  • Toca ☁️ → ⚙️ → Escribe un PIN y toca "Guardar"
  • Los datos se cifran antes de subirse a la nube
  • Usa el mismo PIN en todos tus dispositivos
  • Si olvidas el PIN, no podrás recuperar los datos de la nube

❓ Preguntas frecuentes

¿Firebase es gratis?

Sí. El plan Spark (gratuito) incluye 1GB de almacenamiento en Realtime Database y 10GB de transferencia mensual. Para uso personal es más que suficiente.

¿Necesito tarjeta de crédito?

No. Puedes usar Firebase con el plan gratuito sin ingresar datos de pago.

¿Qué pasa si no configuro Firebase?

La app funciona perfectamente sin Firebase. Todos los datos se guardan localmente. Solo necesitas Firebase si quieres sincronizar entre dispositivos o tener respaldo en la nube.

¿Puedo usar backup JSON sin Firebase?

Sí. Los botones "Exportar Backup" e "Importar Backup" funcionan sin Firebase. Generan un archivo .json que puedes guardar donde quieras.

¿Las reglas de prueba expiran?

Sí, las reglas de "modo de prueba" expiran en 30 días. Ve a Realtime Database → Reglas y actualízalas con las reglas que se muestran en el Paso 2.

¿Mis datos están seguros?

Los datos se guardan en tu proyecto personal de Firebase (solo tú tienes acceso a la consola). Para mayor seguridad, activa un PIN de cifrado dentro de la app — así los datos se cifran antes de subirse.

¿Puedo hospedar la app sin XAMPP?

Sí. Puedes subir los archivos a GitHub Pages (gratis) o cualquier hosting estático. La app no necesita servidor backend — todo funciona en el navegador.
